프로미스는 완료(실패하든 성공하든)될 것으로 기대되는 비동기 작업을 나타내는 객체로, 다음과 같이 생성합니다.


const asyncTask = new Promise((resolve, reject) => {
  try{
    setTimeout(() => resolve('resolved'), 500);
  } catch(e) {
    reject(e);
  }
});

asyncTask
  .then(result => console.log(result))
  .catch(e => console.log(e));

for(let i = 0; i < 10; i++) {
  console.log(i);
}
// 0 ~ 9 숫자가 출력된 후 프로미스가 resolve 됨


프로미스 생성자 내부에서 setTimeout 함수를 통해 500ms 이후에 프로미스가 resolve(=성공) 됩니다. resolve되기 이전에 에러가 발생한다면 프로미스는 reject(=실패)됩니다. 위의 예제 코드에서 프로미스 객체인 asyncTask는 500ms 내에 성공(resolve) 또는 실패(reject)할 것을 '약속'합니다. 아직 완료되지 않은 프로미스의 상태는 pending 상태라고 합니다.


then 함수에서 사용하는 인자 result는 resolve 함수를 통해 전달한 값입니다. 마찬가지로 catch 함수에서 사용하는 인자 e는 reject를 통해 받은 값입니다.
